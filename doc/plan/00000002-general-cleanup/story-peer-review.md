# Peer Review — General Cleanup

## Summary

This is a general review of the existing codebase as of `main` (post merge of the ML learning package, PR #2), covering `game_engine_core`, `game_engine_learning`, and both examples. It was cross-checked against the two earlier peer reviews (`00000000-initial-design`, `00000001-add-ml-package`) so it does not re-report items already resolved or consciously deferred there. The core algorithmic logic (outcome-perspective conventions, MCTS select/expand/evaluate/backpropagate, self-play target labelling) was checked deliberately and is correct for the TicTacToe path; the findings below are packaging, search-quality, protocol-contract, and consistency issues.

## Tooling recommendations

Not code defects, but gaps in the development setup that let several of the above slip through:

- **pyright** — the codebase is heavily and carefully typed (PEP 695 generics, `Literal` conventions, documented Pylance workarounds), but no type checker runs outside the author's editor, so findings #5 and #6 are latent. Add `pyright` as a dev dependency with a minimal `[tool.pyright]` section in `pyproject.toml`, and run it in CI or as a pre-commit step. *(Installation was deferred during this review, so #5/#6 are from manual analysis — a first full run may surface additional findings.)* **Deferred:** CLI pyright requires a node runtime, unwanted on the host machine; planned to land in a future story that containerizes the development environment. Pylance provides interim type checking in the IDE.
- **pytest** — required by #4. **Deferred:** goes with the #4 test-suite story; `.gitignore` already anticipates it.
- **ruff** — optional, but would mechanically enforce the CONTRIBUTING.md import conventions (relative-within-package, no deprecated `typing` aliases — catching #10) via the `TID` and `UP` rule groups. `.gitignore` already anticipates `.ruff_cache/`. **Deferred:** to the same containerization story as pyright.

**Notes verified as correct (no action):** the outcome-perspective conversion in `StandardGame.run` (terminal outcome negated into an absolute result via `active_player_id`); the PUCT sign convention (`exploitation = -average_value` scores a child from the parent's perspective); terminal-node handling in select/expand/evaluate (terminal nodes are never marked fully expanded, are re-evaluated with their exact outcome on each visit, and backpropagate correctly); `SelfPlayCollector` target-value signs for the alternating-play case; `_visit_distribution`'s inclusion of zero-visit legal plies and its uniform fallback; the `str(ply) → column` mapping in `tictactoe_policy_loss`; and the eval/train mode handoff between `NeuralNetworkEvaluator.evaluate_position` and `TrainingLoop.train`.
