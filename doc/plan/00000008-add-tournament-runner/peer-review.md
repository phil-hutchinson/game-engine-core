# Peer Review — Add Tournament Runner

## Summary

Adds a game-agnostic round-robin tournament layer to `game_engine_core` (`tournament/`: runner, result models, standings/cross-table aggregation, markdown + per-game JSON reporting), checkpoint naming/discovery helpers in `game_engine_learning`, a `--checkpoint-every` option in the learning example's `train.py`, and a `tournament.py` example that pits checkpoints against random and heuristic-MCTS reference players. The implementation follows the story and all seven plan steps (including the README check) with no material deviations; the one addition beyond the plan — a `--temperature` CLI flag on `tournament.py` — is justified since without it extra games per pairing are deterministic repeats. `pyright`: 0 errors. `ruff check .`: clean. Full test suite passes (90 tests).
